Features

Agent profiles

Configure Codex, Claude, OpenCode, OpenClaude, Grok-compatible tools, and custom CLIs.

Embedded terminal

Run local shells and agent CLIs from one focused interface.

Project context

Open a folder, inspect files, and generate project-aware context.

Context injection

Send context through clipboard, prompt files, or stdin with confirmation.

Guided installation

Show install commands clearly and require confirmation before running them.

Session history

Track sessions and optional transcript previews locally.

Local-first secrets

Per-tool environment variables stay local and are passed only to selected tool processes.

What SuperTerminal Is Not

  • A replacement for Codex, Claude, OpenCode, OpenClaude, Grok, or any other agent.
  • A distributor or bundled package of third-party CLI tools.
  • A cloud agent platform.
  • An API key broker.
  • A replacement for your existing terminal.
  • A tool that silently installs software or runs commands without confirmation.

Security & Privacy

SuperTerminal is local-first.

  • SuperTerminal does not bundle third-party CLI tools.
  • SuperTerminal does not manage cloud accounts or OAuth.
  • API keys are stored locally and only passed to the selected tool process when configured by the user.
  • Project context is not uploaded by SuperTerminal.
  • Tool installation commands are shown clearly and require user confirmation.
  • SuperTerminal does not intentionally include API keys in prompts, prompt files, diagnostics, or transcript previews.
